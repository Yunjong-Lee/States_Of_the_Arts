- product : 2 stage E2E system (24년초, CN 양산 개시) → 1 stage E2E systme (26.09, VW_EU SOP)
  + Software Definde Approach (소프트웨어 정의 접근법) 기반 개발 : SoC에 종속되지 않고 다양한 플랫폼에 SW Release 가능

  + Technical Features  
    * 기존 운전자 지원 시스템 아키텍처 : 규칙 기반(rule-based) 시스템 (센싱 → 인식 모듈 → 센서 정보 융합 → 융합 결과 기반 주행 전략(planning) 수립)  
    * 보쉬의 운전자 지원 시스템 아키텍쳐 : AI 기반 End to End 학습 중심 + 구조화 (하이브리드 형태, 기존 규칙 기반 시스템에서 사용 중인 인식, 융합, 주행 전략 수립 모듈은 존재하지만 이 모듈들을 훈련하는 방식에서 차이가 발생)  
      - 실제 도로에서 문제 발생 시 문제를 추적·검증하고 안전하게 형식 승인을 받으려면 "구조"가 필요하며, 그 다음 단계로 end-to-end 스택 안에 생성형 AI 결합
        + 스택의 중간 계층에 VLM(Vision Language Model or Vision Language Architecture) 추가를 통해 semantic information(의미적 정보) 생성
  + 한계점
    * 생성형 AI 모델에는 스케일링의 한계(limits of scaling), 발현(emergence), 정렬(alignment) 문제란 세 가지 측면의 한계가 있다
    * 지도학습의 경우, 알고리듬 훈련에 필요한 데이터의 양은 정의할 수 없고, 강화학습 모델의 동작을 보장하기 위해서는 무한한 검증 케이스가 필요하다.
    * 생성형 AI 모델의 경우도 descrete token을 기반으로 작동하므로, continuous한 일상의 세계에 대해서 풀기 어려운 문제가 존재한다.
